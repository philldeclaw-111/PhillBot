# Handoff Methodologies — PhillBot

Purpose: Ensure consistent handoffs between sessions and agents.

Status: Drafted; enforcement required.
Open Items / Remaining Work:
- Define what constitutes a milestone for this project.

---

## Methodology A — System State Snapshot

Include:
- Current baseline versions (Docker, Compose, base image, `gh`).
- Active branch and commit.
- Build/run status (image built? container running?).
- Key runtime behavior notes.

---

## Methodology B — Open Items and Risks

Include:
- Known issues with impact and repro (see known_issues.md).
- Open items with owners and dates (see status_tracker.md).
- Risks and mitigations.

---

## Methodology C — Test Evidence

Include:
- Test plan status (Pass/Fail/Pending) from test_plan.md.
- Evidence links (logs, screenshots, command output).
- Unvalidated paths.

---

## Methodology D — Decision Log

Include:
- Major decisions made this session.
- Option matrix references.
- Architecture stamp references (architecture.md).

---

## Methodology E — Next Steps

Include:
- Ordered task list for the next session.
- Required validations before new work.

---

## Methodology F — Rule Sync

Include:
- Any new guardrails added.
- Confirmation that the Global Rule Pack repo was updated (if applicable).
